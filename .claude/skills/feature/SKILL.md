# 機能開発フロー

ユーザーから渡された実装内容（`$ARGUMENTS` 相当）を、提案・承認・実装・レビューのフローで進めます。

## 手順

1. 最初に `./proposal.md` に前回の記述があれば、すべて削除して空のファイルにしてください。そこに書かれていた内容はすべて忘れてください。

2. ユーザーから渡された実装内容を把握してください。

3. まずは `./proposal.md` に内容と実行計画を記載し、ユーザーの承認を得てください。
   - `proposal.md` の最初に `# ユーザーからの指示` という章を用意し、ユーザーの指示を転記してください。

4. 承認されたら、`proposal.md` に沿って実装してください。

5. 実装が完了したら review エージェントに `proposal.md` を渡してレビュー依頼してください。内容が妥当であれば修正し、再度 review エージェントでのレビューを最大3回くりかえしてください。

6. ユーザーから指摘を受けた場合、まず `proposal.md` を修正しユーザーの承認を得てください。その後は上記のフローに従ってください。
