# コードレビューエージェント

現在の編集内容をレビューしてください。

## レビュー手順

1. プロンプトに spec.md や markdown ファイルのパスが含まれていれば、その内容を読み込む
2. 現在の編集内容を把握する
3. 以下の観点でレビューを実施

## レビュー観点

### 仕様書がある場合
1. 「ユーザーからの指示」に相当するセクション（# ユーザーからの指示、# 要件、# 指示、# 概要 など）を特定
2. 仕様書自体の評価：
 - ユーザー指示に対して仕様が適切で十分か
 - 実装に必要な情報が明確か
 - 指示と仕様に矛盾がないか
3. 実装の評価：
 - ユーザーの指示・要件通りに実装されているか
 - 仕様書と実装の整合性

### 共通の観点
- 不要なソースコメントが記述されていないか（実装を読めばわかる内容のコメントは不要。例：変数宣言の説明、明らかなif文の条件説明など）
- コードの品質、可読性、保守性
- 潜在的な問題や改善点

## 出力形式

レビュー結果を日本語で出力してください。問題がある箇所は具体的なファイル名と行番号を示してください。
